# Dockerfile

1. `FROM`: 底层镜像
2. `RUN`: 构建容器内运行的命令
3. `COPY`: 复制宿主机文件到容器中
4. `ADD`: 同上，过程能够自动解压，且能够复制网络资源
5. `EXPOSE`: 声明开放端口（只做申明，不会做映射，只是方便程序员看的）
6. `ENV`: 设置环境变量
7. `CMD`: 容器启动时执行的命令（只能执行一条）
8. `WORKDIR`: 声明容器内工作目录

## 基于Dockerfile自定义镜像

1. 先创建一个用来放置`Dockerfile`和内置文件的文件夹

编辑`Dockerfile`文件，模版如下：

```dockerfile
# ===================== 1. 基础镜像（按需替换）=====================
# 选择官方稳定版基础镜像，推荐指定具体版本（避免latest导致版本不一致）
# 示例：Python -> python:3.10-slim，Node.js -> node:18-alpine，Java -> openjdk:17-jre-slim
FROM python:3.10-slim as builder

# ===================== 2. 构建环境配置 =====================
# 设置工作目录（容器内的目录，统一管理应用文件）
WORKDIR /app

# 设置环境变量（避免Python缓冲输出、时区、编码等）
ENV PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    TZ=Asia/Shanghai \
    LANG=C.UTF-8 \
    LC_ALL=C.UTF-8

# 安装系统依赖（按需添加，如编译依赖、工具等）
# 注意：alpine镜像用apk，debian/ubuntu镜像用apt-get
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    libpq-dev \
    && apt-get clean && rm -rf /var/lib/apt/lists/*

# ===================== 3. 依赖安装（分层构建，优化缓存）=====================
# 先复制依赖文件（利用Docker缓存，依赖不变则无需重新安装）
COPY requirements.txt .

# 安装Python依赖（示例，其他语言替换为对应命令，如npm install、mvn package）
# --no-cache-dir：不缓存安装包，减小镜像体积
RUN pip install --no-cache-dir -r requirements.txt

# ===================== 4. 复制应用代码 =====================
# 复制当前目录所有文件到容器工作目录（排除.dockerignore中的文件）
COPY . .

# ===================== 6. 运行时配置（安全&权限）=====================
# 创建非root用户（避免容器以root运行，提升安全性）
RUN groupadd -r appgroup && useradd -r -g appgroup appuser

# 更改工作目录权限为非root用户
RUN chown -R appuser:appgroup /app

# 切换到非root用户
USER appuser

# ===================== 7. 暴露端口（声明性，非强制映射）=====================
# 声明容器要暴露的端口（仅文档作用，实际映射需用docker run -p）
EXPOSE 8000

# ===================== 8. 健康检查（监控容器状态）=====================
# 定期检查容器是否正常运行，失败则Docker可自动重启（按需调整检查命令）
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# ===================== 9. 启动命令（推荐用exec形式，避免shell层）=====================
# 方式1：前台运行（适合Web应用，如FastAPI/Flask）
CMD ["gunicorn", "--bind", "0.0.0.0:8000", "app:app"]

# 方式2：后台服务（如需启动多个进程，建议用supervisord，示例如下）
# RUN apt-get update && apt-get install -y supervisor && rm -rf /var/lib/apt/lists/*
# COPY supervisord.conf /etc/supervisor/conf.d/
# CMD ["supervisord", "-c", "/etc/supervisor/supervisord.conf"]
```

2. 创建`.dockerignore`文件

添加需要忽略的文件

```
# 版本控制文件
.git
.gitignore
```

3. 构建镜像

```shell
docker build -t [镜像名称]:[版本标签] [构建上下文路径]
```

- `[镜像名称]`：自定义镜像名（如 `my-nginx`、`python-app`）；

- `[版本标签]`：可选，如 `v1`、`latest`、`1.0`（默认 `latest`）；

- `[构建上下文路径]`：指定 Dockerfile 所在的目录（最常用 `.` 表示当前目录）。
