# 内存CPU资源管理

Docker 可以通过**运行时参数**（临时生效）和**Compose 文件**（持久化）两种方式，限制容器的 CPU、内存使用，防止容器占用过多宿主机资源。

## CPU分配规则

给容器分配不要超过总核心数的**70%-80%**

## 内存分配规则

给容器分配的内存不要超过可用内存的**70%**

```shell
# 查看主机配置
lscpu | grep "CPU(s)"
free -h
```

## 配置

1. 命令行

```shell
docker run -itd \
  --name <container_name> \
  --cpus 2 \       # CPU 核心数
  -m 2g \          # 内存上限
  --storage-opt size=10G # 存储上限，不包括-v挂载的内容
  <image:tag>
```

2. `docker-compose.yml`模版

```yaml
version: '3.8'

services:
  app-1:
    image: 你的镜像
    container_name: app-1
    cpus: 1
    mem_limit: 2g
    restart: always

  app-2:
    image: 你的镜像
    container_name: app-2
    cpus: 1
    mem_limit: 2g
    restart: always

  # 一直写到 app-40 ...
```

**启动流程**：

1. 后台启动

```shell
docker-compose up -d [服务名]
```

2. 容器控制

```shell
# 容器停止
docker-compose stop
# 容器删除
docker-compose down
```

3. 状态查看

```shell
# 查看容器状态
docker-compose ps
# 查看资源占用
docker stats
# 日志
docker-compose logs -f
```
