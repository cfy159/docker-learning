# 本地镜像仓库

## Registry

用于快速搭建、仅需基础存储与推送拉取的场景。

- 开放端口默认为5000

### 快速启动示例

```shell
# 拉取官方 registry 镜像
docker pull registry

# 启动 registry 容器（默认端口 5000）
docker run -d -p 5000:5000 --name <container_name> register

# 推送本地镜像到私有 registry
docker tag <image:tag> localhost:5000/<image:tag>
docker push localhost:5000/<image:tag>

# 从私有 registry 拉取
docker pull localhost:5000/<image:tag>
```

### 查看私有仓库信息

```shell
# 查看具体镜像
curl http://你的仓库IP:5000/v2/_catalog
# {"repositories":["ubuntu","myapp","nginx"]}

# 查看镜像版本
curl http://你的仓库IP:5000/v2/镜像名/tags/list
# {"name":"myapp","tags":["v1","v2","latest"]}
```

### 完全配置

**前置步骤**

```shell
### 1. 生成ssl
# 创建目录
mkdir -p /opt/registry/certs
# 生成证书（直接复制整段运行）
openssl req -x509 -newkey rsa:4096 -nodes -sha256 \
  -keyout /opt/registry/certs/registry.key \
  -out /opt/registry/certs/registry.crt \
  -days 3650 \
  -subj "/CN=你的服务器IP"
# 生成 /opt/registry/certs/registry.crt（公钥） 和 /opt/registry/certs/registry.key（私钥）
 
### 2. docker一键生成账号密码
# 创建目录
mkdir -p /opt/registry/auth
# 生成账号密码（user 是账号，123456 是密码，可自己改）
docker run --rm --entrypoint htpasswd httpd:2 -Bbn user 123456 > /opt/registry/auth/htpasswd
```

```yaml
### 3. 配套config.yml
# 这个文件是私有docker的仓库的总配置文件（自定义）
version: 0.1 # 配置文件的版本信息
log: # 日志配置 让仓库输出运行日志，方便排查问题。
  fields:
    service: registry
storage: # 存储配置（**最重要**）
  delete:
    enabled: true # 允许删除镜像
  filesystem:
    rootdirectory: /var/lib/registry # 镜像存储（在容器中）位置
http: # 这段用来配置https证书在（在容器中）位置
  addr: :5000
  tls:
    certificate: /certs/registry.crt # 证书公钥
    key: /certs/registry.key # 证书私钥
auth: # 账号密码认证（安全核心）开启登录验证docker login
  htpasswd:
    realm: basic-realm
    path: /auth/htpasswd # 账号密码文件路径
```

**容器启动脚本**

```shell
docker run -d \
  -p 5000:5000 \
  --name registry \
  --restart=always \ # 服务器重启、容器意外退出都会自动恢复
  --health-cmd "curl -f --cacert /certs/registry.crt https://localhost:5000/v2/" \ # 容器内部访问自己的 HTTPS 接口
  --health-interval=10s \ # 每 10 秒 检查一次健康状态
  --health-retries=3 \ # 连续失败 3 次 才判定容器不健康
  -v /opt/registry/data:/var/lib/registry \ # 所有上传的 docker 镜像都存在宿主机 /opt/registry/data
  -v /opt/registry/certs:/certs \ # 把宿主机证书文件夹映射到容器内，用于 HTTPS 加密
  -v /opt/registry/auth:/auth \ # 挂载 账号密码认证文件夹
  -v /opt/registry/config.yml:/etc/docker/registry/config.yml \ # 挂载自定义配置文件
  -e REGISTRY_STORAGE_DELETE_ENABLED=true \ # 允许删除仓库里不用的镜像
  registry:2
```

### 删除步骤

1. 确保开启的删除功能

```yaml
storage:
  delete:
    enabled: true
```

2. 查看镜像和版本

```shell
curl -u "user:123456" https://你的服务器IP/v2/_catalog
curl -u "user:123456" https://你的服务器IP/v2/<image>/tags/list
```

3. 执行删除

```shell
curl -u "账号:密码" -X DELETE https://<服务器IP>/v2/<镜像名>/manifests/$(
curl -u "账号:密码" -I -H "Accept: application/vnd.docker.distribution.manifest.v2+json" \
https://<服务器IP>/v2/<镜像名>/manifests/<标签> | grep Docker-Content-Digest | awk '{print $2}'
)
```

4. 设置垃圾回收

上面删除其实并没有删除，还需要通过垃圾回收的机制去清理

```shell
docker exec <container_name> registry garbage-collect /etc/docker/registry/config.yml
```

- `registry`：内部程序
- `garbage-collect`：把标记为删除的文件删除
- `/etc/docker/registry/config.yml`：告诉垃圾回收程序，**按照哪个配置文件进行清理**，只有根据这个文件中配置去进行清理

## Harbo

### 安装部署

1. 安装

```shell
cd /usr/local
tar -zxvf harbor-offline-installer-v2.10.0.tgz
cd harbor
```

2. 修改配置文件

```shell
cp harbor.yml.tmpl harbor.yml
```

修改内容：

```sh
# 1. 修改主机名（你的服务器IP/域名）
hostname: 192.168.1.100   # 改成你自己的服务器IP

# 2. 关闭 HTTPS（测试环境，生产建议开启）
# https:
  # port: 443
  # certificate: /your/cert.pem
  # private_key: /your/key.pem
```

3. 安装

```shell
./install.sh
```

4. 访问`Harbor Web`

- 地址：`http://你的服务器IP`
- 默认账号：`admin`
- 默认密码：`Harbor12345`

### Docker连接Harbo

Docker 默认只信任 HTTPS 仓库，**本地 HTTP 仓库需要配置信任**。

1. 修改`docker`配置

```shell
vim /etc/docker/daemon.json

{
  "insecure-registries": ["http://192.168.1.100"]
}
```

2. 更新配置

```shell
systemctl daemon-reload
systemctl restart docker
```

3. 启动服务

```shell
cd /usr/local/harbor
sudo docker-compose up -d
```

4. 登录本地仓库

```shell
docker login http://192.168.1.21
# admin
# Harbor12345
```

5. `push`镜像

```shell
# 打标签
docker tag <本地镜像名:标签> <Harbor地址/项目名/镜像名:标签>
docker push 192.168.1.21/library/nginx:v1 # library是Harbor默认项目名
```
