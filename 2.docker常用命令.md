# docker常用命令

## 镜像

```shell
# 查询本地镜像
docker images
# 删除本地镜像
docker rmi <image ID> # 一个或多个 image

# 搜索镜像
docker search <image>
# 显示 offical 的是官方镜像。

# 下载镜像
docker pull <image>
```

## 容器

```shell
# 创建容器
docker run -itd [-p <服务器端口:docker端口>] --name <容器名> <image:tag>
# -i: 交互式，保持标准输入
# -t: 为容器分配一个tty终端
# -d: 容器启动后台运行

# 删除容器
docker rm <container_id>
# 删除所有的 container
docker rm `docker ps -aq` 

# 查看容器
docker ps [-a][-q]
# -a: 该选项可以查出未启动的容器
# -q: 静默，只输出id

# 容器控制
docker start/stop/restart <container>

#进入正在运行的 docker 容器
docker exec -it [container_id] /bin/bash

# 创建软链接
docker tag <image:tag> <new_image:new_tag>
# 本地和容器文件传输
docker cp <container_id>:/<file> /target_dir
```

## 打包

将容器打包成镜像主要有两种方式：**docker commit（快照式打包）** 和 **docker export（文件系统导出）**。

### commit操作

1. 快照

```shell
docker commit [选项 参数] <容器ID/容器名> <新镜像名>:<标签>
# -a：指定作者（如 -a "cfy <cfy@example.com>"）
# -m：提交说明（如 -m "Installed nginx & custom config"）
# -p：提交时暂停容器（默认 true，避免数据不一致）
```

2. 导出

```shell
docker save <image:tag> -o <压缩包>.tar
```

3. 导入

```shell
docker load -i <压缩包>.tar
```

### export操作

1. 导出容器

```shell
docker export <容器ID/容器名> -o <输出文件名>.tar
```

2. 导入为镜像

```shell
docker import <压缩包>.tar <image:tag>
```
