# Docker的安装与配置

- 添加安装需要的支持

  ```bash
  #安装必要支持
  sudo apt install apt-transport-https ca-certificates curl software-properties-common gnupg lsb-release
  ```

- 添加阿里云GPG Key

  ```bash
  curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
  ```

- 添加apt源

  ```bash
  #Docker官方源
  echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  
  #阿里apt源
  echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  ```

- docker的安装

  ```bash
  #安装最新版本的Docker
  sudo apt install docker-ce docker-ce-cli containerd.io
  ```

- docker安装验证

  ```bash
  #查看Docker版本
  sudo docker version
  
  #查看Docker运行状态
  sudo systemctl status docker
  ```

- docker命令补全工具

  ```
  sudo apt-get install bash-completion
  
  sudo curl -L https://raw.githubusercontent.com/docker/docker-ce/master/components/cli/contrib/completion/bash/docker -o /etc/bash_completion.d/docker.sh
  
  source /etc/bash_completion.d/docker.sh
  ```

- 配置docker使用国内镜像源

  ```bash
  #Docker首次安装后需要手动创建daemon.json配置文件
  cd /etc/docker
  vim daemon.json
  
  #daemon.json中内容如下
  {
      "registry-mirrors": [
      	"https://docker.mirrors.ustc.edu.cn",
          "https://registry.docker-cn.com",
          "http://hub-mirror.c.163.com",
          "https://mirror.ccs.tencentyun.com",
  		"https://docker.xuanyuan.me",
  		"https://阿里云镜像加速实例获取.mirror.aliyuncs.com"  
      ],
      "dns" : [
      "114.114.114.114",
      "8.8.8.8"
    ],
    "proxies": {
      "http-proxy": "http://YOUR IP:YOUR PORT",
      "https-proxy": "http://YOUR IP:YOUR PORT"
    },
    "exec-opts": ["native.cgroupdriver=systemd"],
    "log-driver": "json-file",
    "log-opts": {
      "max-size": "200m"
    }
  }
  
  #配置完成后重启docker服务
  #清理缓存
  docker system prune -a
  #重新获取配置信息
  systemctl daemon-reload 
  #重启docker
  systemctl restart docker
  ```

  按照上述步骤即可对Docker使用国内镜像源，但是，部分镜像仍然需要前往官方中央Hub拉取，所以最佳实践是在宿主Windows系统上开加速器，并配置代理，在上面替换对应的IP和端口号后可以通过Windows系统中的加速器进行中央仓库的访问。
