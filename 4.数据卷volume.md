# Docker Volume数据卷

`Docker Volume`（数据卷）是 Docker 专门用来持久化容器数据的工具，核心作用：**让容器里的数据脱离容器生命周期独立存在，容器删了、重建了，数据还在**。

**功能**

1. **数据持久化**
2. **数据共享**
   - 多个容器之间共享同一份数据
   - 宿主机 ↔ 容器之间方便读写文件
3. **数据管理与迁移**
   - 备份、恢复、迁移数据非常方便，不用进入容器操作。

## 命名卷流程

1. 创建数据卷

```shell
docker volume create <volume_name>
```

2. 查看所有的数据卷

```shell
docker volume ls

# 详细信息: 卷在宿主机上的真实路径
docker volume inspect <volume_name>
```

3. 把数据卷挂载到容器

```shell
docker run -itd \
  --name <container_name> \
  -v <volume_name>:/container/path \  # 卷名:容器内路径
  <image:tag>
```

4. 删除数据卷

```shell
# 删除单个卷
docker volume rm <volume_name>

# 清理所有不用的卷
docker volume prune
```



## bind-mount流程

将宿主机中的文件、目录 mount 到容器上。

和volume的区别就是无需提前创建数据卷。

1. 直接运行容器时挂载

```shell
docker run -itd \
  --name <container_name> \
  -v /host/path:/container/path \  # 宿主机路径:容器路径
  <image:tag>
```

2. 热挂载

```shell
# 把本地项目挂载到容器
docker run -itd -v /home/your-project:/usr/share/nginx/html nginx
```
