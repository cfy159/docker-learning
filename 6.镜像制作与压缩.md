

# 镜像制作与压缩

## 打包

将容器打包成镜像主要有两种方式：**docker commit（快照式打包）** 和 **docker export（文件系统导出）**。

### commit操作

1. 快照

```shell
docker commit [选项 参数] <容器ID/容器名> <新镜像名>:<标签>
# -a：指定作者（如 -a "cfy <cfy@example.com>"）
# -m：提交说明（如 -m "Installed nginx & custom config"）
# -p：提交时暂停容器（默认 true，避免数据不一致）
```

2. 导出

```shell
docker save <image:tag> -o <压缩包>.tar
```

3. 导入

```shell
docker load -i <压缩包>.tar
```

### export操作

1. 导出容器

```shell
docker export <容器ID/容器名> -o <输出文件名>.tar
```

2. 导入为镜像

```shell
docker import <压缩包>.tar <image:tag>
```

模版：

```shell
docker import --change 'CMD ["/usr/sbin/sshd","-D"]' <压缩包>.tar python_base_test:final
```



## 镜像压缩

### docker-squash工具

该工具能够将commit快照中所有层都压缩到一层中，减小镜像大小

```shell
sudo apt install python3-pip
pip install docker-squash

# 配置环境
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 清理系统中的缓存

打开系统容器后清理缓存文件

```shell
apt-get clean
apt-get autoremove -y
rm -rf /var/lib/apt/lists/* /tmp/* /var/tmp/* /var/log/* /usr/share/doc/* /usr/share/man/* /root/.cache
```
